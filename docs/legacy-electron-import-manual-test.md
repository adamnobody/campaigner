# Ручная проверка импорта из Campaigner 0.2.0

Эта проверка требует реальную базу Campaigner 0.2.0. Не сливать PR без прохождения сценария на такой базе.

## Подготовка

1. Сохранить резервные копии старой и новой БД:
   - `%APPDATA%\campaigner\campaigner.sqlite`
   - `%APPDATA%\com.campaigner.app\campaigner.sqlite`
2. Подготовить старую БД и uploads:
   - старая БД лежит в `%APPDATA%\campaigner\campaigner.sqlite`
   - старые файлы лежат в `%APPDATA%\campaigner\uploads`
3. Убедиться, что новая БД пустая по проектам:
   ```sql
   SELECT COUNT(*) FROM projects;
   ```
   Для первого запуска импорта результат должен быть `0`.

## Сценарии

1. Запустить Campaigner 0.3.0.
2. Убедиться, что появился диалог импорта с количеством проектов, персонажей, фракций и заметок.
3. Нажать `Спросить позже`, перезапустить приложение и убедиться, что диалог появился снова.
4. Нажать `Пропустить`, перезапустить приложение и убедиться, что диалог больше не появляется.
5. Восстановить пустую новую БД из резервной копии или удалить новую БД для повторного чистого прогона.
6. Запустить приложение, нажать `Импортировать`, дождаться отчёта и перезагрузить приложение.
7. Проверить новую БД:
   ```sql
   SELECT COUNT(*) FROM projects;
   SELECT COUNT(*) FROM characters;
   SELECT COUNT(*) FROM factions;
   SELECT COUNT(*) FROM notes;
   SELECT * FROM schema_migrations WHERE name = 'legacy_electron_import';
   PRAGMA foreign_key_check;
   ```
   `PRAGMA foreign_key_check` должен вернуть пустой результат.
8. Проверить в UI:
   - проекты открываются
   - персонажи, фракции, карты и заметки видны
   - изображения персонажей, фракций, династий, карт и обложек открываются
   - заметки с кириллицей, эмодзи и спецсимволами отображаются корректно
   - scenario branches и branch overrides работают
9. Перезапустить приложение ещё раз и убедиться, что диалог импорта больше не появляется.
